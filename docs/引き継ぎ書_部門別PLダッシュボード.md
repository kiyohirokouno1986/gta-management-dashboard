## 0. これは何か

GTAの部門別管理会計（部門別PL）を、目標（10期計画）と実績（部門別PL）を連動させて毎月レビューするための社内ダッシュボード。ユニットリーダーMTGと経営会議で使う。

- 「部門別PLが実績だけ」だと争点が経費配賦に偏るため、目標と連動させて経営と現場のベクトルをそろえるのが狙い。
- 各部門を「投資／収益」で区分し、評価軸を変える。配賦（間接費の按分）も明確なルールに基づく。

実装は現状2系統で同じ画面を出している：
1. Cowork アーティファクト（個人閲覧・コネクター経由でシート取得）
2. Google Apps Script ウェブアプリ（組織限定・社内共有用）← Claude Codeで主に踏襲するのはこちら

## 1. 全体アーキテクチャ

```
[データ源]
 ① 10期計画＆実績シート（Googleスプレッドシート）  → 目標・売上計画・営業利益計画（＝LIVE）
 ② 部門別PL（Excel／管理会計）                    → 利益段・配賦人月の実績（＝SNAP, スナップショット）

[サーバー側（Apps Script Code.gs）]
 doGet() が ① の2タブを読み、LIVE(plan/act) を作って HTML テンプレート Page.html に注入

[クライアント（Page.html / 単一HTML）]
 注入された LIVE ＋ 埋め込み SNAP を使って、JSでDOMを描画
 外部依存なし（ライブラリ不使用、SVGとdivで描画）

[配信]
 Apps Script ウェブアプリ（実行＝自分／アクセス＝組織内）→ URLをGoogle Sitesに埋め込み
```

ポイント: **売上・計画は①からライブ取得（シート編集で自動反映）。利益段・配賦人月は②のスナップショットを埋め込み（毎月、新しい部門別PLで更新→再デプロイ）。** これは「部門別PLがExcelのみでクラウドに無い」ための折衷。将来②もシート化すれば全自動化できる。

## 3. データ構造（JS）

### LIVE（①から生成）
```
LIVE = { plan: 2次元配列, act: 2次元配列 }
// plan[i] = シート(10期計画) (i+3)行目の [2月,3月,…,1月,通期]（千円の生値）
// act[i]  = シート(10期実績) 同上
// 取得ヘルパ:
rowVals(b, R)   // b[R-3] の月次4つ（2〜5月）を ×1000 円で返す
rowVals12(b, R) // 同 12ヶ月
rowExpr(b,'12+25') // '+'対応（注文＝12+25）
salesPlan(u)    // ユニットの売上計画（plan_sales/12+25対応）
```

### SNAP（②を埋め込み）
```
SNAP = {
 consul:{uri:[…4ヶ月],gp,dc1,dc2,dctot,kou,ind,bu,jin:[…]},
 auka:{…}, chumon:{…}, fudosan:{…}, cx:{…}, zen:{…合計…}
}
// uri=売上高 gp=売上総利益 dc1=直接コスト① dc2=直接コスト② dctot=直接コスト計
// kou=貢献利益 ind=間接コスト bu=部門利益 jin=配賦人月（小数）
// snapUnit(key, metric): zen=合計, saas=chumon+fudosan+cx の合算, それ以外はそのまま
```

### UNITS（ユニット定義）
```
zen(全社), saas(SaaS全体), chumon(SaaS注文), fudosan(不動産仲介),
cx(CX), auka(マケプレauka), consul(コンサル)
// 各: name, tag(区分表示), plan(売上計画の行 or '12+25'), brk(売上内訳の[ラベル,行]配列)
```

### TARGETS（目標の定義）★ここが評価ロジックの核
```
zen:     営業利益＝10期計画ライン（kind:'op_line' → LIVE.plan 4行目）
saas:    部門利益＝計画売上×5%（plan_margin, m:'bu', rate:0.05）
chumon:  部門利益＝計画売上×40%
fudosan: ①売上の計画達成（plan_sales）②貢献利益の赤字≤▲200万（flat, th:-2,000,000）
cx:      ①貢献利益≥0（flat, 黒字キープ）②貢献利益＝計画売上×25%（plan_margin）
auka:    貢献利益＝計画売上×50%
consul:  部門利益＝計画売上×45%
// 月次目標 = その月の計画売上 × 目標率（事業計画の成長に連動）。率は初期値で要調整。
```

## 4. 画面構成（タブと各ユニットパネル）

### タブ
- ユニット: 全社／SaaS全体／SaaS注文／不動産仲介／CX／auka／コンサル
- 進捗: 全社の売上・営業利益の「計画 対 実績」を 月次折れ線＋累計進捗グラフ（SVG）で表示
- 目指す姿: 現状（実績のみ→経費配賦が争点）→あるべき姿（目標・実績シートと部門別PLの連動）→投資/収益で役割が変わる、の構想図
- ルール・方針: 配賦ルール全文（目的・配賦5項目・目標の決め方・前提）

### ユニットパネルの並び（上から）
1. ヘッダー: ユニット名・区分タグ・配賦人月（大きく）
2. このユニットの扱い（分配ルール・方針）: 区分／配賦人月／目標利益率／評価軸／特記
3. 目標カード: 目標①②を大きく、当月（5月）と累計（2〜5月 YTD）の達成を表示
4. 今月のアドバイス（自動分析）: 数字からロジックでコメント生成
5. 目標×月マトリクス: 行＝目標・売上内訳、列＝2〜1月、過去＝達成色（緑/黄/赤）・未来＝計画ペース目標（紫）・今月枠
6. 部門別PLの利益段（月次テーブル）: 売上高〜部門利益を 2〜5月＋累計 で表示
7. 利益構造（MQ）: 今月（5月）＋累計（2〜5月）の2本（変動費／限界利益／固定費／利益）

### 主要関数（Page.html）
- `parseSheet` / `rowVals` / `rowVals12` / `rowExpr` / `salesPlan` / `snapUnit` / `sumUnits`
- `targetSeries(u,t)` … 目標の月次配列(tgt12)と実績配列(act)とisRateを返す（評価の共通化）
- `goalRow` / `buildMatrix`(matRow,matCell) / `plBox`(mqBar) / `advicePieces`/`adviceCard`
- `unitRuleCard` / `rulesPanel` / `visionPanel` / `progressPanel`(svgChart)
- `render`（タブ生成と分岐）, `load`（LIVE取得 or 注入→render）

## 5. 業務ルール（実装に埋め込む決めごと）

- **配賦キー**: 間接費は配賦人月で按分。配賦前のプール総額と配賦後の各部門負担を両方見せる。
- **アルバイトの配賦（最重要）**: アルバイトスタッフは**何名でも「分配人数1」**でカウント（人数連動をやめる）。CXは **アルバイト分（何名でも1）＋ アルバイト以外の山内・降旗（2名）＝合計3人月（固定）**。理由＝アルバイト費用は直接経費（人件費）に計上済みで、頭数を数えると間接費まで二重に効くため。
- **部門の括り**: 注文・仲介・コンサルCXは SaaS全体のグロスで見る。
- **投資／収益の区分**: 投資部門＝赤字許容（達成率・売上で評価）／収益部門＝黒字（部門利益）で評価。四半期で見直す。
- **評価の二段**: ①直接（貢献利益）で黒字か → ②配賦後（部門利益）で黒字か。
- **目標の決め方**: 売上＝10期計画。利益＝計画売上×目標率（注文40/コンサル45/auka50/CX25/SaaS5%）。全社営業利益＝10期計画ライン。
- **前提**: 費用は詳細＋証憑を5日までに経理へ提出。

## 6. デプロイ（Apps Script ウェブアプリ・組織限定）

1. script.google.com で新規プロジェクト → `Code.gs` を貼り付け。
2. HTMLファイル「page」を作成 → `Page.html` を貼り付け（**ファイル名は大小区別。Code.gs の `createTemplateFromFile('page')` と一致させる**）。
3. デプロイ→ウェブアプリ／実行＝自分／アクセス＝組織内の全員。
4. 出たURLをGoogle Sitesに「埋め込み」。サイトも組織限定に。

`Code.gs` の要点（シート読み取り→LIVE注入）:
```js
function doGet(){
  var ss=SpreadsheetApp.openById('15ybH2lLFLpV0DqgeVWawsrArhKcL8jHdC_Yp7eQ5dhI');
  var live={plan:gridFor_(ss,'10期計画'), act:gridFor_(ss,'10期実績')};
  var t=HtmlService.createTemplateFromFile('page'); t.liveJson=JSON.stringify(live);
  return t.evaluate().setTitle('ユニットMTGダッシュボード').addMetaTag('viewport','width=device-width, initial-scale=1');
}
function gridFor_(ss,prefix){ // prefixで始まるタブの 3行目以降 × H〜T列
  var sh=ss.getSheets().filter(function(s){return s.getName().indexOf(prefix)===0;})[0];
  var v=sh.getDataRange().getValues(), out=[];
  for(var r=2;r<v.length;r++){var row=v[r],c=[];for(var k=7;k<=19;k++)c.push(row[k]==null?'':row[k]);out.push(c);}
  return out;
}
```
Page.html 側は先頭に `<script>window.__LIVE__ = <?!= liveJson ?>;</script>` を置き、本体JSは `LIVE = window.__LIVE__` を使う（Cowork版は `window.cowork.callMcpTool` でmarkdownを取得して同形に変換していたが、Apps Script版は注入で置換済み）。

## 7. 月次の更新運用

- 売上・計画: 10期シートを編集 → ページ再読込で自動反映（ライブ）。
- 利益段・配賦人月: 新しい部門別PL（Excel）から `SNAP` を再生成 → Page.html を貼り替え → 再デプロイ。
- 表示の最新月: `LATEST`（0=2月…3=5月）を最新月に更新。SNAPに月を足せば自動で選択肢化（旧UIの月セレクタは現状非表示）。
- 配賦人月の新ルール（アルバイト固定）の数値反映: 桑原さん作成の更新版・部門別PLで配賦人月を直してから SNAP に取り込む（現状CXは3.75のまま＝旧値。ルール文言は3人月固定を明記済み）。

## 8. 制約・既知の課題・次の一手

- 部門別PLがExcel依存（クラウド未連携）→ 利益段はスナップショット。②をGoogleシート化すれば全自動・常時最新にできる（推奨）。
- 計画の利益は「計画売上×目標率（初期値）」。各部門の確定利益率が出たら率を差し替え（TARGETS の rate）。
- 財務会計（試算表）と管理会計（部門別PL）の基準差。必要なら全社の実績を財務会計ベースに切替も可。
- 進捗グラフは全社のみ。ユニット別にも展開可能（売上計画/実績・利益で同じ`svgChart`を流用）。
- KPI（MRR・登録率・人件費率など）は別管理。投資部門は達成率・売上で評価する方針のため、KPIはダッシュボード外。

## 9. Claude Code での進め方（推奨）

- 技術スタック自由だが、配信先がGoogle Sites（組織限定）である点が肝。**Apps Script ウェブアプリ構成を踏襲**するのが最短。clasp（Apps Script CLI）でローカル管理＋Gitすると Claude Code と相性が良い。
- ディレクトリ案: `src/Code.gs`（doGet/gridFor_）、`src/page.html`（テンプレート）、`src/app.js`（描画ロジックを分離してテスト可能に）、`data/snap.json`（部門別PLスナップショット）、`config/`（UNITS/TARGETS/UNIT_RULES/行マッピング）。
- まずやること: ①本書の「行マッピング」「SNAP構造」「TARGETS」「業務ルール」をconfig化 → ②Code.gsでシート取得 → ③描画ロジックを関数単位で移植（goalRow/buildMatrix/plBox/advicePieces/progressPanel）→ ④Nodeでロジックの単体テスト（数値の突き合わせ）→ ⑤clasp deploy（組織限定）→ ⑥Sites埋め込み。
- **検証の勘所**: 数値はNodeでスナップショットテスト（例: SaaS注文の部門利益2〜5月、累計、達成率を固定値と照合）。配賦人月・目標率を変えたら全月再計算される設計に。
