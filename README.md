# GTA 部門別PL運用ダッシュボード

GTA の部門別管理会計（部門別PL）を、**目標（10期計画）と実績（部門別PL）を連動**させて毎月レビューするための社内ダッシュボード。ユニットリーダーMTGと経営会議で使う。

Cowork / Apps Script で運用していた版（→ `legacy/`）を、**Vite + TypeScript の SPA** として移植したものです。計算ロジックを型付き・テスト可能なモジュールに分離し、表示は正本の出力を忠実に再現しています。

> 移植の方針: まず**忠実移植**（挙動・数値・見た目を正本と一致）。新機能（部門別PLの自動取込・全ユニット進捗グラフ・目標率の設定UI 等）は後続フェーズで追加します。ロードマップは末尾参照。

## クイックスタート

```bash
npm install
npm run dev        # 開発サーバー（Vite）
npm run test       # 数値ロジックの単体テスト（Vitest）
npm run typecheck  # 型チェック
npm run build      # dist/ に静的ビルド
npm run preview    # ビルド成果物のプレビュー
```

`base: "./"`（相対パス）でビルドするため、`dist/` は任意の静的ホストや iframe 埋め込みで動きます。

## アーキテクチャ

```
[データ源]
 ① 10期計画＆実績シート（Google スプレッドシート）→ 目標・売上計画・営業利益計画（LIVE）
 ② 部門別PL（管理会計 Excel）                     → 利益段・配賦人月の実績（SNAP, スナップショット）

[このリポジトリ]
 src/data/live.json … ① の plan/act グリッド（gridFor_ と同形に整形した埋め込みスナップショット）
 src/data/snap.json … ② の利益段スナップショット
 src/lib/*          … 型付きの純粋ロジック（テスト対象）
 src/views/*        … 表示（HTML文字列を返す純関数。正本の出力を忠実再現）

```

データ取得は `src/lib/data.ts` の `loadData()` に集約（**境界=seam**）。現状は埋め込みJSONを返すだけですが、ここを差し替えれば 10期シートのライブ取得や部門別PLの自動取込へ移行できます。

## データ構造

### LIVE（① 10期シート由来）
`{ plan: GridRow[]; act: GridRow[] }`。各 `GridRow` は `[2月..1月, 通期]`（13列・**千円の生値**）。
`index = シート行 − 3` で参照（例: シート6行目＝`plan[3]`）。`rowVals` 系が ×1000 して円にする。

主な行マッピング（売上・利益）:

| 指標 | 行 |
|---|---|
| 営業利益（全社） | 4 |
| 合計売上高 | 6 |
| SaaS全体（ALL GRIT売上） | 8 |
| SaaS注文 | 12 ＋ 25（注文住宅＋BPO） |
| SaaS不動産仲介 | 18 |
| SaaSコンサル・CX | 30（内訳 コール代行=31 / コンサル・動画=35） |
| マケプレ auka | 39（内訳 自社=40 / 提携TL=44） |
| コンサル（河野） | 48 |

> 5行目（〜前年同月〜）は前年データなので使用しない。当年は6行目。

### SNAP（② 部門別PL由来）
基底6部門 `consul / auka / chumon / fudosan / cx / zen` × メトリクス。各メトリクスは月次配列（現状2〜5月の4ヶ月）。

`uri`=売上高 / `gp`=売上総利益 / `dc1`=直接コスト① / `dc2`=直接コスト② / `dctot`=直接コスト計 / `kou`=貢献利益 / `ind`=間接コスト / `bu`=部門利益 / `jin`=配賦人月（小数）。**値は円。**

### TARGETS（目標＝評価ロジックの核）
- `zen` 営業利益＝10期計画ライン（`op_line`）
- `saas` 部門利益＝計画売上×5%、`chumon` 40%、`consul` 45%（`plan_margin`/`bu`）
- `auka` 貢献利益＝計画売上×50%、`cx` 25%（`plan_margin`/`kou`）
- `fudosan` ①売上の計画達成（`plan_sales`）②貢献利益の赤字 ≤ ▲200万（`flat`）
- `cx` ①貢献利益 ≥ 0（`flat`）②計画ペース（上記）

> 月次目標 = その月の計画売上 × 目標率。率は初期値（`src/config/units.ts` で調整可能）。

## 業務ルール（実装に埋め込み）

- **配賦キー**: 間接費は配賦人月で按分。配賦前プール総額と配賦後の各部門負担を両方見せる。
- **アルバイトの配賦**: アルバイトは何名でも「分配人数1」。CX は アルバイト分（1）＋山内・降旗（2名）＝**3人月固定**。
- **部門の括り**: 注文・仲介・CX は SaaS全体のグロスで見る。
- **投資／収益の区分**: 投資＝赤字許容（達成率・売上で評価）／収益＝黒字（部門利益）で評価。四半期で見直す。
- **評価の二段**: ①直接（貢献利益）で黒字か → ②配賦後（部門利益）で黒字か。
- **基準差の注意**: 売上＝請求書発行月、費用＝決済月。財務会計（試算表）とは基準が異なり数値は完全一致しない。

## 検証

数値ロジックは Vitest でスナップショット的にピン留めしています（`src/lib/logic.test.ts`）。例: SaaS注文の部門利益 2〜5月・累計 15,388,744、不動産仲介の売上達成率、全社売上の累計達成率 99% など。`src/views/render.test.ts` は全ユニットパネルが例外なく描画され、想定の文言・数値を含むことを確認します。

配賦人月・目標率を変えたら全月が再計算される設計です（`agg` / `targetSeries`）。

## 月次の更新運用

- **売上・計画**: `src/data/live.json` を最新の10期シートから再生成（将来はライブ取得に置換）。
- **利益段・配賦人月**: 新しい部門別PL（Excel）から `src/data/snap.json` を再生成。
- **最新月**: `LATEST` はスナップショット末尾月に自動追従（`makeCtx`）。SNAP に月を足すと最新月が進む。
